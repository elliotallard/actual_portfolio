Technical Writing:
Introduction to the Terminal



In this solo project, I created technical documentation to using the terminal - for a complete beginner. Requiring nothing but an understanding of Finder (Mac's default file manager), the instructions use a narrative, employ a user's existing mental model, and encourage experimentation to ensure a reader's basic understanding of the terminal.

Throughout my writing of it, I learned how & when to employ proven methods in delivering technical concepts to a non-technical audience. Some examples of these are - using examples (ha), chunking information, using a narrative, and leveraging existing audience knowledge.
